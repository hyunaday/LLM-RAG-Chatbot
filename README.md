### 💡 아이디어 배경: 왜 RealView인가?

기존 부동산 플랫폼들은 단순 **거래 금액 데이터**만 제공하여, 사용자가 '이 지역이 지금 어떤 상태이고, 앞으로 어떻게 변할 것인지'에 대한 통찰(인사이트)을 얻기 어려웠습니다. 특히 지역 커뮤니티나 소셜 미디어에서 오가는 **'정성적 분위기'** 정보는 파악이 불가능했습니다.

RealView는 이 문제를 해결하기 위해 **RAG(Retrieval-Augmented Generation) 챗봇**을 활용하여 공공 데이터는 물론, 최신 뉴스, 지역 커뮤니티 정보까지 분석하여 사용자에게 **대화형으로 생생하게** 지역 분위기를 전달합니다.

### ✨ 핵심 기능 (MVP)

| 기능 분류 | 상세 내용 | 데이터 활용 |
| :--- | :--- | :--- |
| **1. 지역 시세 및 심층 분석** | 국토교통부 실거래가, 통계청 인구/가구 통계 기반 시세, 평당가 변동 추이, 전세가율 분석 결과 제공 | **공공 데이터 API** |
| **2. RAG 기반 분위기 분석** | 주요 언론사 뉴스, 지역 커뮤니티(맘카페 등) 데이터를 수집/분석하여 **최신 이슈, 개발 호재, 긍정/부정 분위기** 요약 | **뉴스/커뮤니티 검색, RAG** |
| **3. 자연어 질의응답** | "아이 키우기 좋은 동네인가요?", "최근 뜨는 카페는 어디일까요?" 등 **정성적 질문**에 대한 친절한 맞춤형 응답 | **LLM & RAG** |
| **4. 지역 비교 분석** | 관심 지역 간 시세, 인프라, 학군, 교통 등 다각적인 비교 데이터를 AI가 이해하기 쉽게 요약 | **통합 데이터 분석** |

### 🛠️ 기술 스택 및 시스템 아키텍처

#### 주요 기술 스택

  * **백엔드 프레임워크:** `FastAPI` (Python)
  * **프론트엔드/UI:** `Vue.js` (사용자 친화적 웹 챗봇 인터페이스)
  * **LLM/RAG:** `OpenAI RAG Module` & `LangChain`
  * **데이터 파이프라인:** `웹 크롤링` (뉴스/커뮤니티), `공공데이터포털 API` (국토교통부, 통계청)

-----

### 🌐 RealView 시스템 데이터 흐름 (System Data Flow)

RealView 챗봇이 사용자의 질문에 응답하는 처리 과정입니다.

1.  **사용자 입력**
    * **경로:** `사용자 (모바일/웹)` $\rightarrow$ **프론트엔드 (Vue.js 챗봇 UI)**
    * **내용:** 사용자의 **질의 (Query)**가 발생합니다.

2.  **백엔드 수신 및 분류**
    * **경로:** 프론트엔드 $\rightarrow$ **백엔드 (FastAPI Server)**
    * **처리:** 백엔드는 질문을 수신하고, 필요한 정보 유형을 분류합니다.

3.  **데이터 처리 및 RAG 실행 (핵심 단계)**
    * **담당:** **데이터 처리 및 RAG 모듈**
    * **정량 데이터:**
        * **요청:** $\rightarrow$ `공공데이터 API` (국토교통부 실거래가, 통계청 데이터)
    * **정성 데이터 (분위기):**
        * **요청:** $\rightarrow$ `뉴스/커뮤니티 검색 엔진` 및 `외부 데이터베이스`
    * **응답 생성:** 모든 취합된 데이터를 바탕으로 $\rightarrow$ **OpenAI RAG 모듈/LLM**을 통해 최종적인 **인사이트가 담긴 답변**을 생성합니다.

4.  **최종 응답 전달 및 표시**
    * **경로:** RAG 모듈의 응답 $\rightarrow$ 백엔드 서버 $\rightarrow$ **프론트엔드**
    * **결과:** 프론트엔드가 사용자 화면에 답변을 **명확하게 표시**합니다.
